컨텍스트 전달 방식
에이전트 간 메모리 없음 — 매 호출마다 결과를 명시적으로 전달해야 함
⚠ 핵심 제약
Claude는 API 호출 간 메모리가 없습니다. 에이전트끼리 "직접 대화"하는 게 아니라, 항상 코드(또는 n8n)가 이전 결과를 다음 호출의 메시지에 포함시켜 전달합니다.
✓ 해결 패턴
각 서브에이전트 호출 시 system prompt에 역할을, user message에 "이전 에이전트 결과 + 새 지시"를 함께 담아 전달. 히스토리 배열로 대화 맥락을 누적 관리.
// 컨텍스트 체이닝 패턴 const research = await callAgent("리서처", task); const analysis = await callAgent("분석가", `원래 태스크: ${task} 리서처 결과: ${research} ← 이전 결과를 명시적으로 포함 위 내용을 분석하세요.` ); const report = await callAgent("오케스트레이터", `태스크: ${task} 리서처: ${research} 분석가: ${analysis} 최종 보고서를 작성하세요.` );
3
순차 vs 병렬 실행 패턴
서브태스크 간 의존성이 없으면 Promise.all로 동시에 실행해 시간을 단축
순차 실행 (Sequential)
A → B → C 순서로 하나씩 실행. 앞 단계 결과가 다음 단계의 입력이 되는 경우 사용.
const r1 = await agent1(task); const r2 = await agent2(r1); const r3 = await agent3(r2);
병렬 실행 (Parallel)
독립적인 서브태스크를 동시에 실행. 전체 시간 = 가장 오래 걸리는 에이전트 시간.
const [r1, r2, r3] = await Promise.all([ agent1(task), agent2(task), agent3(task) ]);

Git Worktree × 멀티에이전트 개발
각 AI 에이전트가 독립 브랜치에서 병렬로 작업 — 충돌 없이 동시 개발
Git Worktree란?
하나의 Git 저장소에서 여러 브랜치를 동시에 다른 폴더에 체크아웃하는 기능. 브랜치 전환 없이 각 브랜치가 독립된 작업 디렉토리를 가짐.
# 에이전트별 worktree 생성 git worktree add ../agent-research feature/research git worktree add ../agent-analysis feature/analysis git worktree add ../agent-writer feature/writer
멀티에이전트와 결합하면?
각 AI 에이전트(Claude Code 인스턴스)를 별도 worktree 폴더에서 실행하면, 서로 다른 파일을 동시에 수정해도 충돌이 발생하지 않음.
충돌 없는 병렬 작업 독립 컨텍스트 PR로 통합
Git Worktree × 에이전트 팀 타임라인
main (통합)
← PR merge
agent/research
조사 완료
agent/analysis
분석 완료
agent/writer
작성 중
각 에이전트가 독립 브랜치에서 동시에 커밋 → PR로 main에 통합. 에이전트 간 파일 충돌 없음.

단일 vs 멀티에이전트 — 언제 쓸까
복잡도보다 병렬 가능성이 더 중요한 판단 기준
기준 단일 에이전트 멀티에이전트
작업 규모소~중형대형 · 복잡
단계 독립성순차 의존병렬 가능
컨텍스트 요구량단일 윈도우 내초과 시 유리
구현 복잡도낮음높음
비용낮음에이전트 수만큼 증가
디버깅단순오류 전파 추적 필요
적합 예시요약, Q&A, 코드 리뷰RFP 분석, 대규모 리서치

AgentFit 개발 → 배포 흐름
Claude.ai 아티팩트 → Claude Code → GitHub → 웹 서비스
1
Claude.ai 아티팩트로 프로토타입
지금처럼 대화로 빠르게 HTML/JS 프로토타입 제작. API 호출 테스트도 claude.ai 환경에서 즉시 가능.
2
Claude Code로 로컬 프로젝트화
HTML 파일을 받아 폴더 구조 분리 (index.html, styles.css, app.js). Claude Code에게 "기능 추가해줘" 명령으로 이어서 개발.
mkdir maeval && cd maeval # Claude Code 터미널에서 claude "이 HTML을 React 프로젝트로 분리해줘"
3
GitHub 푸시
Git 저장소 초기화 후 GitHub에 업로드. 팀 협업 시 Git Worktree로 에이전트별 브랜치 분리.
git init && git add . && git commit -m "init: AgentFit prototype" git remote add origin https://github.com/yourname/maeval.git git push -u origin main
4
GitHub Pages / Vercel 배포
GitHub Settings → Pages → main 브랜치 선택으로 즉시 무료 배포. 또는 Vercel/Netlify 연결 시 push마다 자동 배포.
