에이전트 워크플로우

에이전트 루프(Agentic Loop)는 AI가 단순히 응답을 생성하는 것을 넘어, 스스로 계획을 세우고 실행하며 결과를 검증하는 자율 반복 구조입니다. 기존 루프 방법론(PDCA, OODA 등)과 개념은 유사하지만, 루프를 돌리는 주체가 사람에서 AI로 전환된다는 점이 핵심 차이입니다.

기존 루프 방법론 vs 에이전트 루프

항목 기존 루프 (PDCA 등) 에이전트 루프 (Claude)
실행 주체 사람 AI
중간 판단 사람이 매 단계 개입 AI가 자율 처리
루프 속도 일/주/월 단위 초/분 단위
외부 시스템 접근 사람이 직접 수행 파일·앱·API 등 자동 접근
오류 발생 시 사람이 수정 후 재시도 AI가 자체 재시도

자체 판단의 실체 — 시스템 프롬프트 구조

에이전트의 자체 판단은 특별한 별도 알고리즘이 아닙니다. LLM의 추론 능력과 Anthropic이 설계한 시스템 프롬프트, 그리고 Tool Call + 컨텍스트 누적 구조의 결합으로 동작합니다.

① 시스템 프롬프트

Anthropic이 설계한 내장 프롬프트로 행동 규칙, 사용 가능한 도구 목록, 판단 기준이 정의됩니다. "모호하면 질문해라", "위험한 작업은 확인해라" 같은 규칙이 여기 포함됩니다.

② LLM 추론 능력

Claude 모델이 현재 상황에서 어떤 도구를 어떻게 써야 하는지 텍스트 추론으로 스스로 결정합니다. 이것이 자체 판단의 실체입니다.

③ Tool Call + 컨텍스트 누적

도구 실행 결과가 대화 컨텍스트에 쌓이고, 다음 추론 때 그 결과를 참고합니다. 사람이 메모를 보며 다음 행동을 결정하는 구조와 동일합니다.

⚠️ 한계 (감독 필요)

자체 판단이라도 결국 LLM 추론이기 때문에, 지시가 모호하거나 예외 상황이 많으면 품질이 저하됩니다. 중요 작업 전 체크포인트 확인이 설계에 포함된 이유입니다.
